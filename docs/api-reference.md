# ARES API Reference

> **Base URL:** `http://localhost:8080`  
> All endpoints require `Authorization: Bearer <token>` or `X-API-Key: <key>`
> **except** `POST /auth/token` and `GET /health`.
>
> **Note:** The web dashboard (`/dashboard/*`) has its own authentication
> using the same tokens — see [Dashboard](#dashboard) section.

---

## Authentication

### `POST /auth/token`

Get a JWT access token + refresh token.

**Request:** `application/x-www-form-urlencoded` (OAuth2 password flow)
```
username=alice&password=changeme
```

**Response:**
```json
{
  "access_token":  "eyJhbGci...",
  "refresh_token": "rt-...",
  "token_type":    "bearer",
  "role":          "operator",
  "expires_in":    3600
}
```

---

### `POST /auth/refresh`

Rotate refresh token. Old token is revoked; new access + refresh tokens issued.  
Rate limited per IP (same limit as `/auth/token`).

```json
{ "refresh_token": "rt-..." }
```

---

### `POST /auth/logout`

Revoke current access token (adds JTI to blacklist).

---

### `GET /auth/me`

Return current authenticated user info.

---

### `POST /auth/register`

Create a new user. Requires `team_lead` role.

```json
{ "username": "bob", "password": "StrongPass1!", "role": "operator" }
```

---

### `POST /auth/change-password`

Change password for the authenticated user.

```json
{ "current_password": "old", "new_password": "NewPass1!" }
```

---

### `POST /auth/api-keys`

Create an API key for the authenticated user.

```json
{ "name": "CI pipeline key" }
```

**Response:**
```json
{ "key_id": "...", "raw_key": "ares_...", "name": "CI pipeline key" }
```

> Store `raw_key` immediately — it is not recoverable after this response.

---

### `GET /auth/api-keys`

List API keys for the authenticated user (key values are masked).

---

### `DELETE /auth/api-keys/{key_id}`

Revoke an API key.

## Campaigns

### `POST /campaigns`

Create a new campaign. Requires `operator` role or higher.

```json
{
  "name":          "Corp Red Team Q1",
  "client":        "ACME Corp",
  "scope":         ["10.0.0.0/8", "192.168.1.0/24"],
  "noise_profile": "normal",
  "operator":      "alice"
}
```

**Response:** `201 Created`
```json
{ "id": "abc12345", "name": "Corp Red Team Q1", "status": "created" }
```

---

### `GET /campaigns`

List all campaigns. Supports pagination: `?page=1&per_page=50`

Response headers: `X-Total-Count`, `X-Page`, `X-Per-Page`

---

### `GET /campaigns/{id}`

Get campaign details including confirmed findings count.

---

### `DELETE /campaigns/{id}`

Delete a campaign. Requires `team_lead`.

The delete operation removes the campaign and stored child data for that
campaign, including findings, hosts, credentials, and loot. Use this for lab
cleanup or removing test campaigns from the dashboard.

**Response:** `200 OK`
```json
{ "status": "deleted", "campaign_id": "abc12345" }
```

---

### `GET /campaigns/{id}/findings`

Get confirmed findings for a campaign.  
Optional filters: `?severity=critical&page=1&per_page=50`

---

### `GET /campaigns/{id}/cvss`

Get CVSS score distribution for a campaign.

---

### `GET /campaigns/{id}/diff/{other_id}`

Compare two campaigns — shows new findings, fixed findings, and severity changes.
Useful for verifying remediation between assessments.

---

### `POST /campaigns/{id}/run`

Run an execution plan against the campaign.

```json
{
  "plan":       [{"module_id": "ad.kerberoast", "params": {"dc": "dc01.corp.local"}}],
  "dry_run":    false,
  "noise_profile": "normal"
}
```

**Response:** `202 Accepted`
```json
{ "execution_id": "exec-abc123", "status": "running", "total_modules": 1 }
```

## Modules

### `GET /modules`

List all registered modules with metadata.

```json
[
  {
    "id":          "ad.kerberoast",
    "name":        "Kerberoasting",
    "category":    "ad",
    "description": "Request Kerberos TGS tickets for service accounts",
    "opsec_level": "medium",
    "requires":    ["domain_creds", "spn_list"],
    "outputs":     ["kerberos_hashes"],
    "mitre":       "T1558.003",
    "mitre_list":  ["T1558.003"],
    "source":      "builtin",
    "param_schema": {
      "dc": {
        "type": "string",
        "description": "DC IP or FQDN",
        "required": true,
        "secret": false
      },
      "password": {
        "type": "string",
        "description": "Password",
        "required": true,
        "secret": true
      }
    }
  }
]
```

`param_schema` is generated by `ares/modules/params.py` and is the source of
truth for dynamic dashboard forms.

---

### `POST /modules/{module_id}/run`

Run a single module directly.

```json
{
  "campaign_id": "abc12345",
  "target":      "dc01.corp.local",
  "params": {
    "domain": "CORP",
    "dc":     "dc01.corp.local"
  },
  "dry_run": false
}
```

## Reports

### `POST /reports/{campaign_id}`

Generate a campaign report.

```json
{
  "format":               "html",
  "include_timeline":     true,
  "include_mitre_heatmap": true
}
```

Supported formats: `html` | `pdf` | `json` | `markdown`

**Response:** `200 OK`
```json
{ "filename": "campaign-id_Corp_20260609_1200.html", "format": "html" }
```

### `GET /reports/{campaign_id}`

List generated report files for a campaign. Results are restricted to files
that belong to the campaign and resolve inside the report directory.

### `GET /reports/{campaign_id}/files/{filename}`

Download a generated report file. Filename traversal, absolute paths, encoded
traversal, and symlink escapes are rejected.

## WebSocket

### `WS /ws/campaigns/{campaign_id}/events?token=<token>`

Real-time event stream for a campaign.  
Auth: `?token=<access_token>` query param (WebSocket cannot send headers before connect).

**Events:**
```json
{ "type": "module_start",      "module_id": "ad.kerberoast" }
{ "type": "module_complete",   "module_id": "ad.kerberoast", "findings": 2 }
{ "type": "finding_discovered","severity": "high", "title": "..." }
{ "type": "campaign_status_change", "status": "complete" }
{ "type": "connected",         "campaign_id": "..." }
{ "type": "keepalive" }
```

## Dashboard

The React dashboard is served at `GET /dashboard/`, with SPA fallback for
routes like `/dashboard/campaigns`. The static shell can load without auth so
the login screen works; all sensitive data is fetched from the main authenticated
API routes above.

The dashboard uses `POST /auth/token`, `POST /auth/refresh`, the main REST API,
and `WS /ws/campaigns/{campaign_id}/events?token=<token>`. Legacy
`/dashboard/api/*` and `/dashboard/ws/live` routes are not the primary data
source.

## Error Responses

All errors return structured JSON:

```json
{
  "code":   401,
  "detail": "Not authenticated. Provide Bearer token or X-API-Key.",
  "type":   "authentication_error"
}
```

HTTP status codes:

| Code | Meaning |
|------|---------|
| 400 | Bad request / validation error |
| 401 | Authentication required or token invalid |
| 403 | Insufficient role / scope violation |
| 404 | Resource not found |
| 429 | Rate limit exceeded |
| 500 | Internal server error |
| 503 | Engine not ready |
